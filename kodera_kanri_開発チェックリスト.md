# kodera_kanri 開発チェックリスト

**このドキュメントを使って、仕様通りに実装されているかを画面ごとに順番に検証する。**
**1画面ずつ「実装→確認→修正→次へ」のサイクルで進める。漏れを防ぐ。**

---

## 開発ルール

### ファイルサイズ制限
- **1ファイル最大700行**。超えたらコンポーネント分割する
- ページファイル（page.tsx）はデータ取得＋レイアウトのみ。UIロジックはコンポーネントに切り出す
- 1コンポーネント = 1責務

### コンポーネント分割の基準
```
page.tsx（~100行）     → データ取得、レイアウト組み立て
├── feature-table.tsx（~300行）  → テーブル本体
├── feature-form.tsx（~200行）   → フォーム
├── feature-modal.tsx（~150行）  → モーダル
└── feature-card.tsx（~150行）   → カード
```

### 命名規則
- ページ: `app/(dashboard)/sites/page.tsx`
- コンポーネント: `components/sites/site-list-table.tsx`（ケバブケース）
- フック: `hooks/use-sites.ts`
- 型: `types/database.ts`

## 確認の進め方

各Phaseで以下のサイクルを回す:

1. **Claude Codeに指示**: 「Phase X の YY を実装して」
2. **ブラウザで確認**: チェックリストの項目を1つずつ目視確認
3. **スクショをClaude Codeに送信**: 「この画面で XX が動いていない」
4. **修正**: Claude Codeが修正
5. **再確認**: 全チェック項目がOKになるまで繰り返す
6. **次のPhaseへ**
