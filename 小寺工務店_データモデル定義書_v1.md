# 小寺工務店 データモデル定義書

**バージョン:** v1.1
**作成日:** 2026年3月13日
**対応要件定義書:** v3.1
**DB:** Supabase（PostgreSQL）
**デプロイ:** Vercel（Next.js App Router）

---

## 0. Supabase固有の設計方針

- 認証は **Supabase Auth** を使用。`auth.users` テーブルはSupabaseが自動管理
- アプリ固有のユーザー情報は `public.profiles` テーブルに保持し、`auth.users.id` とFKで紐付け
- Row Level Security (RLS) を全テーブルに適用し、3ロール（管理者/現場スタッフ/作業員）別のアクセス制御を実現
- 手書きサイン画像は **Supabase Storage** の `signatures` バケットに保存。パスをDBに記録
- リアルタイム機能（Supabase Realtime）は配置入力の確認状況リアルタイム表示で使用

### U1: profiles（ユーザープロフィール ※Supabase Auth連携）

Supabase Authの `auth.users` テーブルと1:1で紐付くプロフィールテーブル。認証自体はSupabase Authが管理し、アプリ固有の情報（ロール・作業員紐付け）をここに保持する。

| カラム名 | 型 | NULL | 説明 |
|---------|---|------|------|
| id | UUID | NOT NULL | PK. auth.users.id と同一値 |
| worker_id | INT | NULL | FK → workers.id。作業員と紐付く場合（NULLは管理者専用アカウント等） |
| role | ENUM | NOT NULL | 管理者 / 現場スタッフ / 作業員 |
| display_name | VARCHAR(50) | NOT NULL | 表示名 |
| line_user_id | VARCHAR(100) | NULL | LINE User ID。LINE OAuth連携時に取得 |
| created_at | TIMESTAMPTZ | NOT NULL | |
| updated_at | TIMESTAMPTZ | NOT NULL | |

**備考:**

- `auth.users` テーブルにはメールアドレス・パスワードハッシュ・最終ログイン日時等がSupabaseにより自動管理される
- 新規ユーザー登録時にSupabaseのトリガーで `profiles` レコードが自動作成される
- LINE OAuth連携の場合、`line_user_id` にLINE User IDが格納され、`auth.users` にもOAuthプロバイダとして記録される
- RLSポリシー: 管理者は全テーブルにフルアクセス。現場スタッフも管理者と同等（打刻関連テーブルのみアクセス不可）。作業員は assignment_workers.confirmed のUPDATEのみ可

## 4. データフローとテーブルの対応

```
[ログイン]
  → READ: auth.users（Supabase Auth）→ profiles（ロール判定）
  → セッション: JWT token

[現場一覧（PC）]
  → READ: sites + companies（一覧表示）
  → READ: assignments（本日配置人数の集計）
  → INSERT: sites（新規現場登録。ステータス「稼働中」で作成）
  → UPDATE: sites（全項目編集可能。ステータス「完了」への変更含む）

[画面① 配置入力]
  → INSERT: assignments + assignment_workers + assignment_partners + assignment_locations
  → INSERT: sites（フリー入力の場合、現場マスターに即時登録してからassignments.site_idにリンク）
  → READ: workers, sites(status='稼働中'), companies, partner_companies, location_types

[LINE通知]
  → READ: assignments + assignment_workers → profiles.line_user_id
  → UPDATE: assignment_workers.confirmed

[画面② 日報入力]
  → READ: assignments + assignment_workers（プリセット用）
  → INSERT: daily_reports + report_workers + report_work_categories + report_partners

[画面③ 手書きサイン]
  → INSERT: signatures（画像はSupabase Storageに保存、パスをDBに記録）
  → READ: companies.signer_name

[画面④ 出面表]
  → READ: assignments + daily_reports（突合表示）
  → INSERT/UPDATE: dezura_records
  → UPDATE: daily_reports.check_status

[画面⑤ 出勤表]
  → READ: dezura_records + daily_reports + report_workers + workers + company_calendar
  → INSERT/UPDATE: attendance_monthly

[画面⑥ 経理CSV]
  → READ: attendance_monthly + workers + sites + companies
  → OUTPUT: CSV file
```
