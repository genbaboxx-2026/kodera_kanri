## 1. プロジェクト概要

### 1.1 背景

株式会社小寺工務店では、日々の人員配置（出面＝デズラ）管理・日報・出勤表・請求業務をすべてExcel手入力とLINEテキストで運用している。現場数は1日20〜70件、作業員は40〜150名規模で、以下の課題が顕在化している。

- 白籏さんがLINEで送る予定出面が頻繁に変更され、最新情報がわからなくなる
- 職長の紙日報を写真で送り、事務が毎朝Excel出面表に手入力している（二度手間・三度手間）
- 日報・出面表・出勤表の三つを月末に突合する作業で、不一致やヒューマンエラーが多発
- 経理への転記も手作業（出面表→給与入力Excel→弥生給与）で、請求漏れリスクがある
- マネーフォワード勤怠が形骸化している

### 1.2 プロジェクト目標

- 日報提出をトリガーに出面表・出勤表を自動生成し、手入力作業を大幅に削減する
- 人員配置の最新情報を一元管理し、作業員への通知・確認を仕組み化する
- 月末の突合作業を「差異チェック・承認」のみに軽減する
- 経理向けCSV出力で弥生給与への転記を自動化する

### 1.3 スコープ

**Phase 1（今回）:**

- 人員配置管理機能（白籏さんの出面管理）
- 日報提出・管理機能（職長の日報デジタル化）
- 常用日報のスマホ手書きサイン機能
- 出面表の自動生成
- 出勤表の自動集計
- LINE連携（通知・確認ボタン）
- 事務員の打刻機能
- 経理向けCSV出力（弥生給与連携）
- マスターデータ管理

**Phase 2（将来）:**

- 経理側の請求書発行・会計処理の自動化
- 営業の受注管理・見積もり管理
- 経営ダッシュボード
- 多言語対応（ベトナム語）
- 資材管理部の配車管理連携

| 項目 | 決定事項 |

## 2. ユーザーと役割（3ロール）

| ロール | 該当者 | 人数 | デバイス | 認証方式 |
|--------|--------|------|---------|---------|
| 管理者 | 佐藤さん（事務）、経理、小寺社長 | 3〜5名 | PC | Supabase Auth（メール/PW） |
| 現場スタッフ | 白籏さん（手配）、職長 | 7〜11名 | スマホ（会社貸与） | Supabase Auth（メール/PW） |
| 作業員 | 自社社員、外国人技能実習生 | 40〜150名 | スマホ（個人持ち） | LINE OAuth |

**ロール別のアクセス範囲:**

| ロール | PC画面 | スマホ画面 | LINE |
|--------|--------|----------|------|
| 管理者 | 全機能フルアクセス（現場一覧・出面表・出勤表・マスタ設定・マイページ・打刻） | 配置入力・日報入力 | - |
| 現場スタッフ | 全機能アクセス可（現場一覧・出面表・出勤表・マスタ設定・マイページ）※打刻のみ不可 | 配置入力・日報入力 | - |

## 3. マスターデータ定義

システム稼働前に以下のマスターデータを登録する必要がある。各マスターが利用される画面・機能を明記する。

**登録項目:**

| 項目 | 型 | 必須 | 説明 |
|------|---|------|------|
| 作業員ID | 自動採番 | ○ | システム内部ID |
| 氏名 | テキスト | ○ | 例: 増尾勇次、ハー |
| 氏名カナ | テキスト | ○ | 検索・ソート用 |
| 雇用区分 | 選択 | ○ | 正社員 / 外国人技能実習生 |
| 役職 | 選択 | ○ | 職長 / 一般 / 管理 / 事務 / 経理 / 運転手 |
| 所属 | 選択 | ○ | 仮設事業部 / 管理課 / 技術課 / 経営管理部 |
| 固定残業時間 | 数値 | ○ | 個人別に設定。例: 技術課勤務あり→5h、管理課新入→20h |
| LINE連携ID | テキスト | - | LINE友だち追加時に自動紐付け |
| 在籍ステータス | 選択 | ○ | 在籍 / 退職 |
| Web認証アカウント | 有無 | ○ | 職長以上に発行 |

実習生: ハー、クアン、ダン、フォン、トゥー、チョン、レ、ホップ、ハイ、ター、ルアン、ホアン、ヴィエット、グエン ほか

**利用画面:**

| 画面 | 利用方法 |
|------|---------|
| 配置入力（白籏さん） | 作業員名の選択肢として表示。タップで現場に配置 |
| 日報入力（職長） | 作業者名のプリセット表示。追加・削除の選択肢 |
| 出面表（事務） | 列ヘッダーの作業員名として表示 |
| 出勤表（事務・経理） | 個人詳細のプロフィール表示（氏名・雇用区分・役職・所属・固定残業時間） |

### 3.2 現場マスター

稼働中の全現場を管理する。配置入力で現場を選択する際の選択肢となり、日報・出面表・出勤表に連鎖する。

**重要:** 発注元・支払者は現場マスターに「デフォルト値」として登録する。配置追加時にこの値が自動入力されるが、まれに変わるケースがあるため手動で変更できる。変更した場合、変更後の値は配置データ（assignments テーブル）に保存され、現場マスターのデフォルト値は変わらない。

**登録項目:**

| 項目 | 型 | 必須 | 説明 |
|------|---|------|------|
| 現場ID | 自動採番 | ○ | システム内部ID |
| 現場名 | テキスト | ○ | 例: 銀行協会神田、松戸駅 |
| 工事番号 | テキスト | - | 未定の場合は空欄。後から現場一覧の詳細パネルで編集可能 |
| 発注元 | FK(発注元・支払者マスター) | ○ | デフォルト値。配置追加時に自動入力されるが、まれに変わる場合は手動変更可能 |
| 支払者 | FK(発注元・支払者マスター) | ○ | デフォルト値。発注元と同一の場合も別の場合もある。まれに変わる場合は手動変更可能 |
| 着工日 | 日付 | - | 未定の場合は空欄。後から編集可能 |
| 完工予定日 | 日付 | - | 長期案件は年単位、短期は1〜2日 |
| ステータス | 選択 | ○ | 稼働中 / 完了。新規登録時は「稼働中」。現場一覧の「完了にする」で「完了」に変更 |
| デフォルト契約区分 | 選択 | - | 常用 / 請負。配置入力時の初期値 |
| 備考 | テキスト | - | 現場一覧の詳細パネルで編集可能 |

**初期登録データ例:**

| 現場名 | 発注元 | 支払者 | 備考 |
|--------|--------|--------|------|
| 銀行協会神田 | シンワ | シンワ | 発注元と支払者が同一 |
| 九段南2丁目 | シンワ | シンワ | |
| 松戸駅 | フジタ | フジタ | 常用あり |
| 末吉町 | MJ | MJ | 常用あり |
| 鶴沼高校 | 木原工業 | 木原工業 | 解体工事 |
| 綾瀬THSC | 永沢興業 | 永沢興業 | 常用あり |
| 本郷三丁目 | 三同建設 | 三同建設 | |

**利用画面:**

| 画面 | 利用方法 |
|------|---------|
| 現場一覧（事務・管理者） | 全現場の一覧表示・新規登録・全項目編集・ステータス管理。現場名クリックで詳細パネル展開 |
| 配置入力（白籏さん） | 現場名の選択肢（ステータス「稼働中」のみ表示）。選択すると発注元・支払者・デフォルト契約区分がデフォルト値として自動入力される。フリー入力時はsitesテーブルにINSERTされる |
| 日報入力（職長） | 予定出面からプリセット。現場名が表示され、紐づく発注元・支払者も表示 |
| 出面表（事務） | 行の「現場」「発注元」「支払者」列の値。現場一覧から「この現場の出面表を見る」で遷移 |
| 出勤表（事務・経理） | 個人別の勤務先現場名として表示 |
| 経理CSV出力 | 案件別の集計軸（発注元別・現場別の常用/請負人数） |

**重要:** 現場を選択した時点で発注元・支払者が自動セットされることで、白籏さんの入力負荷を大幅に削減する。

## 4. 画面・機能定義

### 4.0 ログイン画面

**画面の目的:** Supabase Authによるユーザー認証。管理者・現場スタッフはメール/パスワード、作業員はLINE OAuth連携。

**認証フロー:**

| ロール | 認証方式 | フロー |
|--------|---------|--------|
| 管理者・現場スタッフ | メール / パスワード | ログイン画面でメール＋PW入力 → Supabase Auth認証 → ロールに応じた画面へ遷移 |
| 作業員 | LINE OAuth | LINE公式アカウント友だち追加 → 初回のみ社員番号入力で紐付け → 以降はLINE内で操作完結（Web画面への直接ログインは不要） |

**ログイン後の遷移先:**

| ロール | デバイス | 遷移先 |
|--------|---------|--------|
| 管理者 | PC | 現場一覧タブ（PCナビゲーション） |
| 現場スタッフ | スマホ | 配置入力画面 |

**機能:**

- パスワードリセット（メール送信）
- セッション管理（Supabase Auth のJWTトークン）
- ログアウト（マイページから）

### 4.0b 現場一覧（PC・管理者）

**画面の目的:** 全現場を一覧管理する。現場の新規登録・ステータス管理を行う。現場名クリックで該当現場の全情報を編集できる詳細パネルが展開し、出面表にもジャンプできる。

**利用タイミング:** 常時。PCナビゲーションのデフォルトタブ。

**参照するマスター:** 現場マスター / 発注元・支払者マスター

**画面構成:**

| エリア | 内容 |
|--------|------|

**現場テーブルの列:**

| 列 | 内容 |
|----|------|

| 着工日 | |

|-----------|------|------|

**「+ 現場を新規登録」ボタンの動作:**

登録フォームが開く。入力項目:

- 現場名（必須）
- 工事番号（任意。未定の場合は後から追加可能）

- 現場マスターのステータスを「完了」に変更
- 現場一覧では薄く表示される
- 配置入力画面では、完了した現場はマスター検索候補に表示されなくなる（過去の配置データは閲覧可能）

**現場詳細パネル（現場名クリックで展開）:**

| 完工予定日 | ○ | 日付選択 |
| デフォルト契約区分 | ○ | 常用 / 請負 |

詳細パネルの下部にはサマリー（本日配置人数・夜勤人数・協力会社数）と「この現場の出面表を見る」ボタンを配置。ボタンクリックで出面表タブに遷移し、該当現場でフィルタされた状態で表示される。

**配置入力画面との連携:**

### 4.1 画面① 配置入力（白籏さん・スマホWeb）

**画面の目的:** 翌日の全作業員の配置先を一元管理し、LINEで全員に通知する。現行のLINEテキスト手打ち運用を置き換える。過去の配置も参照できる。

**利用タイミング:** 当日13:00頃〜夜にかけて（変更があれば何度でも修正・再配信可能）

**参照するマスター:** 作業員マスター / 現場マスター / 発注元・支払者マスター / 協力会社マスター / 勤務場所区分マスター

**画面構成:**

| エリア | 内容 |
|--------|------|
| ヘッダー | 「配置入力」タイトル + 「前日コピー」ボタン |
| 日付ナビゲーション | ← 8月28日（木） → の矢印で日付を切替。過去の配置は参照専用、未来は編集可能 |
| サマリーバー | LINE配信前: 配置済み人数 / 未配置人数の2項目のみ表示。LINE配信後: 配置済み / 未配置 / 確認済み / 未確認の4項目を表示。配信済みバッジ（配信時刻付き）も表示 |
| 未配置アラート | 赤色で未配置者名を表示（全員が配置されるまで消えない） |
| 現場カード（複数） | 現場ごとのカード。作業員ピル＋協力会社＋/- ＋合計 |
| 「+ 現場を追加」ボタン | 点線ボタン。タップで現場マスターから選択するモーダルが開く |
| 現場以外のカテゴリ | 東京出張所・本社・運搬・置場・休み等に配置された作業員 |
| フッター | 「下書き保存」「LINE配信」ボタン |

**現場カードの構成（1カードあたり）:**

| 要素 | 説明 |
|------|------|
| カードヘッダー | 現場名（太字） / 発注元（サブテキスト） / 契約区分バッジ（請負 or 常用） |
| 作業員ピル | 配置された作業員名がピル形式で並ぶ。「+ 追加」ボタンで作業員マスターから選択して追加 |
| 協力会社行 | 会社名 + -/数値/+ のカウンター。人数を +/- で増減 |
| 「+ 協力会社を追加」 | 協力会社マスターから選択して新しい行を追加 |
| カードフッター | 計 N名 / 夜勤バッジ（夜勤者がいる場合） |

**入力項目（現場カードごと）:**

| 項目 | 型 | 入力方法 | 説明 |
|------|---|---------|------|
| 現場名 | FK | マスターから選択 | 「+ 現場を追加」で追加。選択すると発注元が自動セット |
| 発注元 | FK | 自動入力 | 現場マスターから取得。カード上ではサブテキスト表示 |
| 支払者 | FK | 自動入力 | 現場マスターから取得。必要に応じて手動変更可 |
| 契約区分 | 選択 | 常用 / 請負 | 現場マスターのデフォルト値がセット。タップで切替 |
| 作業員名 | FK複数 | マスターからタップ選択 | 「+ 追加」ボタンから。配置済みの人はグレーアウト |
| 勤務区分 | 選択 | 日勤のみ / 通し夜勤 / 夜勤のみ | 作業員ごとに設定可能 |
| 協力会社名 | FK | マスターから選択 | 「+ 協力会社を追加」から |
| 協力会社人数 | 数値 | +/- カウンター | ボタンで増減。最小1 |
| 備考 | テキスト | 手入力（任意） | 例: 「←常用」等の特記事項 |

**UI上の操作フロー:**

1. 日付ナビゲーションで対象日を確認（デフォルト翌日。← → で過去・未来に移動可能）
2. 「前日コピー」で前日の配置データを複製（差分修正が楽になる）
3. 「+ 現場を追加」ボタンで現場カードを追加。現場マスターから選択すると発注元・契約区分が自動入力
4. カード内の「+ 追加」で作業員をタップ選択して配置
5. 協力会社は「+ 協力会社を追加」で会社を選択、+/- カウンターで人数調整
6. 夜勤がある場合は作業員の勤務区分を変更
7. 東京出張所・本社・運搬・置場・休み等のカテゴリにも作業員を配置
8. 上部のサマリーで「未配置: 0名」を確認
9. 「LINE配信」ボタンで全作業員にLINE通知

**特殊機能:**

- **日付ナビゲーション:** ← → 矢印で過去・未来の配置を閲覧可能。過去の配置は参照専用（編集不可）。当日以降は編集可能
- **前日コピー機能:** 前日の配置データをコピーして対象日分を作成。差分だけ修正すればよい
- **現場追加機能:** 「+ 現場を追加」ボタンで現場追加モーダルが開く。2つのタブがある:

  **タブA: マスターから選択（通常はこちら）**
  1. 現場名を検索（テキスト入力で現場マスターから候補を絞り込み。候補には現場名・発注元・工事番号を表示）
  2. 候補から現場を選択すると、発注元・支払者・工事番号が現場マスターのデフォルト値から自動入力される。発注元・支払者はまれに変わるケースがあるため「変更」リンクから手動変更可能。工事番号は現場に固定のため変更不可
  3. 契約区分を選択（請負 / 常用。現場マスターのデフォルト値がプリセット。変更可）
  4. 勤務区分を選択（日勤のみ / 通し夜勤 / 夜勤のみ。デフォルトは日勤のみ）
  5. 備考（任意。自由記述）
  6. 「この現場を追加」で確定 → 新しい現場カードが追加される（作業員0人の状態）

  **タブB: フリー入力（新しい現場をその場で登録して追加）**
  1. 現場名を自由テキストで入力（必須）
  2. 発注元を発注元マスターから選択（必須。表記揺れ防止のため自由テキスト不可）
  3. 支払者を発注元マスターから選択（必須）
  4. 工事番号（任意。未定の場合は空欄でOK。現場一覧から後で編集可能）
  5. 契約区分を選択（請負 / 常用。必須）
  6. 勤務区分を選択（日勤のみ / 通し夜勤 / 夜勤のみ。必須）
  7. 備考（任意）
  8. 「この現場を追加」で確定 → 現場マスターに即時登録（ステータス「稼働中」）→ 新しい現場カードが追加される → 現場一覧にも即座に表示される
- **協力会社の追加・人数調整:** 各現場カード内で「+ 協力会社を追加」から協力会社マスターを選択。人数は +/- カウンターで調整
- **未配置者アラート:** 全作業員のうちどこにも配置されていない人を赤色でハイライト表示
- **確認状況リアルタイム表示:** LINE配信後のみ、確認ボタンを押した人 / 未確認の人をサマリーバーに表示。配信前は配置済/未配置の2項目のみ
- **18:00アラート:** 入力が完了していない場合、白籏さんにLINEでリマインド

**生成するデータ:** 予定出面データ（日付×現場×作業員の配置テーブル）

**後続への影響:**

- → LINE通知（全作業員へ配置情報を配信）
- → 出面表の「予定」列に反映
- → 日報入力フォームのプリセットデータとなる

### 4.2 画面② 日報入力（職長・スマホWeb）

**画面の目的:** 職長が当日の作業実績を報告する。紙日報を完全に置き換える。予定出面データがプリセットされるため、差分修正だけで済む。

**利用タイミング:** 当日17:00前後（作業終了後）

**参照するマスター:** 作業員マスター / 現場マスター / 協力会社マスター / 作業内容区分マスター

**画面表示時の自動処理:** 予定出面データ（画面①で白籏さんが入力済み）から、当該職長の担当現場の情報をプリセットして表示する。

| プリセットされる項目 | ソース |
|---------------------|--------|
| 現場名 | 予定出面データ |
| 作業者名（複数） | 予定出面データ |
| 契約区分（常用/請負） | 予定出面データ |
| 協力会社名・人数 | 予定出面データ |
| 発注元・支払者 | 現場マスター経由 |

**入力項目:**

| 項目 | 型 | 入力方法 | プリセット | 説明 |
|------|---|---------|-----------|------|
| 日付 | 日付 | 自動 | ○ | 当日日付 |
| 現場名 | FK | 選択 | ○ | 予定出面から。変更可 |
| 記入者 | FK | 自動 | ○ | ログインユーザー（職長） |
| 作業者名 | FK複数 | タップ追加/削除 | ○ | 予定出面から。「+ 追加」で追加・タップで削除 |
| 作業時間（日勤） | 時刻 | 手入力 | ×（8:00〜17:00デフォルト） | 開始〜終了 |
| 作業時間（夜勤） | 時刻 | 手入力 | × | 22:00〜5:00等 |
| 作業人数合計 | 数値 | 自動計算 | - | 作業者数から自動 |
| 契約区分 | 選択 | 常用/請負 | ○ | 予定出面から。変更可 |
| 常用人数 | 数値 | 自動計算 | - | |
| 請負人数 | 数値 | 自動計算 | - | |
| 作業内容区分 | FK複数 | マスターから選択 | × | とび/解体/サポート土工等。複数選択可 |
| 作業内容詳細 | テキスト | 自由記述 | × | 例: 鳥カゴネット畳み墜落防止ロープ盛替(5F) |
| 天候 | 選択 | 晴/曇/雨/雪 | × | 任意 |
| 気温 | 数値 | 手入力 | × | 任意 |
| 風力 | 選択 | 強/中/弱/無 | × | 任意 |
| 欠勤者名 | テキスト | 手入力 | × | 任意 |
| 遅刻早退者名 | テキスト | 手入力 | × | 任意 |

**協力会社セクション:**

予定出面からプリセットされた協力会社が表示される。追加・人数変更・削除が可能。

| 項目 | 入力方法 | 説明 |
|------|---------|------|
| 協力会社名 | マスターから選択 or フリー入力 | 「+ マスターから追加」で協力会社マスターから選択。「+ フリー入力で追加」でマスター未登録の会社名を直接入力（後日管理者がマスター登録） |
| 人数 | +/- カウンター | 会社ごとに人数を増減。表示形式は「蓮見鳶工業 2名」 |

**残業者セクション:**

自社作業員の残業と、協力会社の残業を分けて管理する。

| 項目 | 入力方法 | 説明 |
|------|---------|------|
| 自社残業者名 | 作業者一覧から選択して追加 | 「+ 残業者を追加」で作業者チップから選択。x ボタンで削除 |
| 自社残業時間 | 時刻を手入力 | 残業の開始〜終了を入力。例: 17:00〜17:30。残業時間（0.5h）は自動計算 |
| 協力会社残業 | 会社名を選択して追加 | 「+ 協力会社の残業を追加」で追加。上部の協力会社セクションに登録済みの会社から選択 |
| 協力会社残業人数 | +/- カウンター | 残業した人数を設定 |
| 協力会社残業時間 | 時刻を手入力 | 残業の開始〜終了を入力。残業時間は自動計算 |

**自動計算ルール:**

- 残業時間は0.5時間単位で自動丸め（17:00〜17:30 = 0.5h）
- 作業人数合計 = 作業者名の数
- 常用人数 / 請負人数 = 契約区分に応じて自動集計
- 夜勤（22:00〜5:00）は開始日の出勤としてカウント
- 通し夜勤 → 夜分を1日カウントして × 1.5
- 夜勤のみ → 日勤を欠勤扱い、夜勤手当 × 0.5

**提出時の分岐処理:**

- 契約区分が「常用」→ 提出前に手書きサイン画面（画面③）へ遷移
- 契約区分が「請負」→ そのまま提出完了

**生成するデータ:** 日報データ（実績テーブル）

**後続への影響:**

- → 出面表に自動反映（ステータス: 黄色→ピンク）
- → 常用の場合: サインデータが紐付く
- → 出勤表の集計元データとなる

### 4.3 画面③ 手書きサイン（職長のスマホ画面上で上位会社所属長が署名）

**画面の目的:** 常用作業の場合に、上位会社の所属長からその場でサインをもらい、電子的に保管する。紙の「所属長印」欄を置き換える。

**利用タイミング:** 日報提出時（契約区分が「常用」の場合のみ）。その日のうちに現場で完結。

**参照するマスター:** 発注元・支払者マスター（サイン担当者名のデフォルト値）

**画面仕様:**

| 要素 | 仕様 |
|------|------|

| 入力方式 | 指またはスタイラスで手書き |
| サイン者名 | 発注元マスターのサイン担当者名がデフォルト表示。手動変更可 |

| 確定ボタン | サイン確定。以降は編集不可 |
| タイムスタンプ | 確定時に自動記録（年月日時分秒） |

**生成するデータ:** サインデータ（サイン画像PNG + サイン者名 + タイムスタンプ + 紐付く日報ID）

**保管要件:**

**後続への影響:**

- → 日報データにサイン画像が紐付く
- → 出面表で常用行にサイン済みアイコンが表示される

### 4.4 画面④ 出面表（事務・PC）

**画面の目的:** 予定出面（画面①）と日報実績（画面②）を突合した結果を表示し、事務が確認・承認する。現行のExcel出面表を完全に置き換える。

**利用タイミング:** 毎朝（前日の日報が出揃った後）。翌月18日までに全データを確定。

**参照するマスター:** 作業員マスター / 現場マスター / 発注元・支払者マスター / 協力会社マスター / 勤務場所区分マスター

**データソース:** 予定出面データ（画面①）＋ 日報データ（画面②）をシステムが自動突合。

**表示レイアウト（現行Excel出面表を踏襲）:**

| 列 | 内容 | ソース |
|----|------|--------|
| 日付 | 8月27日等 | 自動 |
| 曜日 | 水等 | 自動 |
| 現場 | 銀行協会神田等 | 現場マスター |
| 発注元 | シンワ等 | 発注元マスター |
| 支払者 | シンワ等 | 発注元マスター |
| 契約区分 | 請負/常用 | 日報データ（予定と異なれば日報を正とする） |
| 特記事項 | 自由記述 | 日報データ |
| 作業員名（複数列） | 増尾、チョン、ハー等 | 日報データ。名前の右に残業時間（0.5等） |
| 協力会社 | 蓮美、加藤等 | 日報データ |
| 夜勤行 | 区切り線の下に別行 | 日報データ |
| 常用人数合計 | 数値 | 自動計算 |
| 請負人数合計 | 数値 | 自動計算 |

**ステータス管理（色分け）:**

| 色 | 意味 | 操作 |
|----|------|------|
| 黄色 | 日報未提出 | 職長に提出を催促 |

| 白色 | 2人目チェック済（確定） | 編集ロック |

**差異検出・アラート（自動）:**

| 差異パターン | 表示方法 |
|-------------|---------|
| 予定出面と日報で作業員が異なる | 該当セルをオレンジ色ハイライト |
| 予定にない作業員が日報に登場 | 「追加」バッジ付きで表示 |
| 予定にいるはずの作業員が日報にいない | 「欠」バッジ付きで表示 |
| 契約区分（常用/請負）が予定と日報で異なる | 区分セルをオレンジ色ハイライト |
| 日報未提出の現場 | 行全体が黄色 |

**事務（佐藤さん）の操作:**

1. 差異ハイライトされた箇所を確認
2. 日報を正としてOKか判断。必要に応じて白籏さんに電話確認
3. 問題なければ「承認」ボタンで1人目チェック完了（黄→ピンク）
4. 2人目がチェックして「最終承認」（ピンク→白）
5. 白になったデータは編集ロック

**生成するデータ:** 確定出面データ

**後続への影響:**

- → 出勤表の集計元（白色データのみが集計対象）
- → 経理CSV出力の元データ

### 4.5 画面⑤ 出勤表（事務・経理・PC）

**画面の目的:** 確定した出面データから個人別の月次出勤表を自動生成する。手書き出勤表・MF勤怠出勤表を完全に置き換える。

**利用タイミング:** 月次。翌月18日までに確定し経理に提出。

**参照するマスター:** 作業員マスター / 現場マスター / 会社カレンダーマスター / 勤務場所区分マスター

**データソース:** 確定出面データ（画面④で白色になったデータ）から個人別に自動集計。

**画面構成:**

| エリア | 内容 |
|--------|------|
| ツールバー | 月選択 / 作業員選択（セレクトボックス）/ 「前の人」「次の人」ボタン / 「CSV出力」ボタン |
| プロフィール | 氏名・雇用区分・役職・所属・固定残業時間 |
| サマリーカード | 出勤日数(日勤)・出勤日数(夜勤)・総労働時間・残業時間・深夜時間・休日出勤の6項目 |
| 日別テーブル | 1日〜月末の日別出勤データ |

**プロフィール表示:**

| 項目 | 内容 |
|------|------|
| 氏名 | 作業員マスターから |
| 雇用区分 / 役職 / 所属 | 作業員マスターから |
| 固定残業時間 | 作業員マスターから |

**サマリーカード（6項目）:**

| 項目 | 計算ロジック |
|------|------------|
| 出勤日数（日勤） | 日勤で出勤した日数 |
| 出勤日数（夜勤） | 夜勤で出勤した日数 |
| 総労働時間 | 日勤＋夜勤の全時間 |
| 残業時間 | 所定内＋所定外の合計 |
| 深夜時間 | 22:00〜5:00の合計時間 |
| 休日出勤 | 法定＋所定の合計日数 |

**日別テーブル:**

| 列 | 内容 |
|----|------|
| 日 | 1日〜月末 |
| 曜 | 土曜は青、日曜は赤で表示 |
| 現場名 | 日勤の現場。通し夜勤は2行（日勤行＋夜勤行） |
| 区分 | 日勤 / 夜勤 |
| 開始 | 作業開始時刻 |
| 終了 | 作業終了時刻 |
| 残業h | 0.5h単位。値がある場合のみ表示 |
| 深夜h | 深夜勤務時間。値がある場合のみ表示 |
| 備考 | 所定休日 / 法定休日 / 通し夜勤(x1.5) 等 |

法定休日の行は赤背景で表示。夜勤行は薄グレー背景。

**機能:**

- 「前の人」「次の人」ボタンで作業員を順に切替
- セレクトボックスで直接指名選択
- 「CSV出力」ボタンで個人のCSVを出力

**夜勤の計算ルール:**

| パターン | 計算 |
|---------|------|
| 通し夜勤（日勤後そのまま夜勤） | 夜分を1日カウント × 1.5 |
| 夜勤のみ（夜勤だけ出勤） | 日勤を欠勤扱い。夜勤手当 × 0.5 |

**固定残業の扱い:**

- 作業員マスターの「固定残業時間」を参照
- 固定残業時間内の残業は所定内残業
- 超過分が所定外残業
- 現場作業員は会議残業のみが固定残業の対象

**生成するデータ:** 月次出勤集計データ

**後続への影響:**

- → 経理CSV出力の元データ

## 6. 非機能要件

### 6.1 利用環境

| 項目 | 要件 |
|------|------|
| 対応ブラウザ | Chrome（PC・スマホ）、Safari（iOS） |
| スマホ対応 | レスポンシブ対応。配置入力・日報入力はスマホファーストで設計 |
| PWA対応 | ホーム画面に追加可能。入力途中データのブラウザキャッシュ |
| 多言語対応 | Phase 1では不要 |
| インターネット接続 | 実習生のスマホはWifi前提。職長は会社貸与スマホでモバイル通信 |
| 同時接続数 | 最大20〜30人程度 |

### 6.2 セキュリティ

- 認証: Supabase Auth。管理者・現場スタッフはメール/パスワード、作業員はLINE OAuth連携
- 認可: Supabase Row Level Security (RLS) で3ロール（管理者/現場スタッフ/作業員）別のアクセス制御
- 管理者: 全機能にフルアクセス（打刻含む）
- 現場スタッフ: 管理者と同等のアクセス権限。ただし打刻機能のみ不可
- 作業員: Web画面にログインしない。LINEで確認ボタンのみ
- 手書きサインデータはSupabase Storageに保存。タイムスタンプ付き・確定後編集不可
- 通信はHTTPS必須（Vercel標準）
- Supabase のJWTトークンによるセッション管理

### 6.3 可用性・保守

- 平日8:00〜翌5:00の稼働を保証（夜勤対応）
- Supabaseの自動バックアップ（日次）
- サインデータ: Supabase Storageに5年保管 / 日報・出面データ: 7年保管
- Vercelの自動デプロイ（GitHubプッシュで自動反映）
